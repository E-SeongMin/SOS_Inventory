## 2. 채택한 개발 기술과 브랜치 전략

### React

- React
    - 컴포넌트화를 통해 유지보수와 재사용성을 고려했습니다.

### 브랜치 전략

- Git-flow 전략을 기반으로 release, develop 브랜치를 사용했습니다.
- release, develop
    - **release** 브랜치는 배포 단계에서만 사용하는 브랜치입니다.
    - **develop** 브랜치는 개발 단계에서 git-flow의 master 역할을 하는 브랜치입니다.
